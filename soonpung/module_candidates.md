# 순풍(順風) — 모듈 후보 비교표 (Phase 0 산출물)

`순풍 = 레짐 필터(언제 탈지) × 변동성 타겟팅(얼마나 탈지) × 헤지 오버레이(어떻게 막을지)`

각 모듈별 후보 기법을 **메커니즘 / 파라미터 / 장점 / 단점 / MVP 채택 여부**로 비교.
MVP는 PBO(과최적화 확률) 억제를 위해 **최소 파라미터**로 레짐 필터 + 변동성 타겟팅 2개만.

---

## 모듈 1 — 레짐 필터 (상승/하락 국면 판별)

| 후보 | 메커니즘 | 파라미터 | 장점 | 단점 | MVP |
|---|---|---|---|---|---|
| **A. SOX 200일 SMA** (Faber⑤) | 기초지수 종가 > 200d SMA → 추세 ON | lookback=200 | 단순·강건, 파라미터 1개, R²=0.97(KORU/EWY 검증) | 횡보장 휩쏘, 후행 | **★ 채택(주)** |
| B. SOX 50/200 정·역배열 | 50MA > 200MA(골든) → ON | 50, 200 | 추세 방향 명확 | 진입 지연 더 큼, 파라미터 2개 | 보조 |
| C. 12개월 TSM 부호 (MOP③) | `sign(R_{252d})` > 0 → ON | lookback=252 | 학술 근거 강함, 자산 자체 모멘텀 | 단독은 변동 큼 | **★ 채택(보조 AND/OR)** |
| D. EWY 200MA | 한국 대용 트렌드 필터 | 200 | KORU 분석 검증 | SOXL과 베이시스 차이 | 후보(앙상블) |
| E. Consensus 다지수 (기존 BUBE) | QQQ/SPY/SMH 3-SMA200 ±2% 합의 | 200, ±2%, dwell | 기존 엔진과 정합 | 복잡, 평균회귀용 튜닝 | 비교 벤치 |

**MVP 결론**: `regime_ON = (SOX > SMA200) AND (TSM_252 > 0)` 또는 OR(둘 중 택1 비교). dwell(이력관성) 적용해 휩쏘 완화.

## 모듈 2 — 변동성 타겟팅 (비중 동적 조절) ★ 순풍의 심장

| 후보 | 메커니즘 | 파라미터 | 장점 | 단점 | MVP |
|---|---|---|---|---|---|
| **A. 실현변동성 역가중** (Cooper①/Harvey④) | `lev = σ_target / σ̂`, σ̂=annualized realized vol | σ_target, win(20/60d), cap | 직관적, decay 방어, MDD↓ | 후행성(급변 한 박자 늦음) | **★ 채택** |
| B. EWMA 변동성 | 지수가중 σ̂ (최근 가중) | λ(≈0.94), σ_target | 반응 빠름 | 노이즈 | 채택(σ̂ 추정 옵션) |
| C. EGARCH (Cooper 원본) | 비대칭 GARCH로 σ̂ 예측 | 다수 | 가장 정밀 | 파라미터 多 → PBO↑ | Phase 3+ |
| D. 목표 변동성 밴드 | 연 30~40% 밴드 밖에서만 조정 | band | 거래비용↓ | 밴드 경계 임의성 | 채택(거래 절감 옵션) |
| E. VVIX/VIX 선행 보조 | 실현변동성 후행 보완 | thresh | 선행성 | 노이즈, 추가 데이터 | Phase 2.5 보조 |

**MVP 결론**: `target_lev = clip(σ_target / σ̂_{60d}, 0, lev_max)`. σ_target ≈ 연 0.30~0.40, lev_max로 풀로딩 상한. 실현변동성 후행성은 VVIX로 후속 보완.

## 모듈 3 — 엔트리 / 피라미딩 (추세 진입·증액)

| 후보 | 메커니즘 | MVP |
|---|---|---|
| MA 정배열 돌파 진입 | 레짐 ON 전환 시 진입 | ★ (레짐 필터에 흡수) |
| 모멘텀 스코어 임계 돌파 | TSM 스코어 > thresh 증액 | Phase 2.5 |
| 추세 강화 단계적 증액(피라미딩) | 변동성 타겟이 자동으로 비중 조절 → 명시적 피라미딩 불필요 | MVP 제외 (vol-target이 대체) |

**MVP 결론**: 별도 엔트리 로직 없이 `position = regime_ON × target_lev`. 피라미딩은 변동성 타겟팅이 사실상 수행.

## 모듈 4 — 디레버리징 / 청산 (하락 선제 대응)

| 후보 | 메커니즘 | MVP |
|---|---|---|
| 레짐 이탈 즉시 축소 | regime OFF → position 0 | ★ 채택 |
| 변동성 급등 자동 축소 | σ̂↑ → target_lev↓ (모듈2가 수행) | ★ 채택(내장) |
| VIX/VVIX 스파이크 트리거 | VIX > thresh → 강제 축소 | Phase 2.5 |
| 트레일링 스톱 + 타임스톱 | 고점 대비 −x% / N일 보유 후 재평가 | Phase 3 |

**MVP 결론**: 레짐 이탈 + 변동성 타겟이 디레버리징을 모두 담당. 명시적 스톱은 검증 후.

## 모듈 5 — 테일 헤지 오버레이 (급락 보험) — Phase 3+

| 후보 | 메커니즘 | 비고 |
|---|---|---|
| SOXS 조건부 소량 | 레짐 OFF & 변동성 급등 시 SOXS 매수 | 상시 헤지는 상승장 드래그 → **조건부 우선** |
| 현금 비중 동적 확대 | target_lev < 1 잔여를 현금 | MVP에서 자연 발생 |
| VIX 콜 대용 | 옵션 가능 계좌 한정 | IBKR 옵션 권한 필요 |

**결론**: 비용 드래그 때문에 **조건부 발동**만. MVP 제외.

## 모듈 6 — 수익 플로어 (누적 수익 보호) — Phase 3+

| 후보 | 메커니즘 | 비고 |
|---|---|---|
| CPPI 쿠션 (⑦) | `risky = m×(P−F)`, F=트레일링 플로어 | 누적 수익 잠금. m·F 파라미터 → PBO 주의 |

**결론**: MVP 검증 통과 후 오버레이. 트레일링 플로어 변형으로 이익 잠금.

## 최종 MVP 사양 (Phase 2 구현 대상)

```
position_t = regime_ON_t × clip(σ_target / σ̂_{60d,t}, 0, lev_max)

regime_ON_t = (SOX_t > SMA200_t)  [AND/OR  sign(R_{252d}) > 0]   # 비교 검증
σ̂_t        = SOXL 실현변동성(60일, 연율화)
σ_target   = 0.30 ~ 0.40 (연)
lev_max    = 1.0 (margin 미사용 — 기존 CHAMP_NOMARGIN 정책 정합)
```

거래 대상: SOXL 1종(MVP). 보조 데이터: SOX/SOXX, VIX, VVIX, EWY, SOXS.
검증(§6): walk-forward, CSCV/PBO, DSR/PSR, **국면별 성과 분해(필수)**, 기존 평균회귀 엔진과 상관·합성 포트 효과.
